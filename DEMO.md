# VoiceCanvas 演示脚本

按此顺序说出指令，可完整展示四个考点：**指令理解准确性、容错性、响应延迟、复杂指令拆解**。

> 准备：后端 + 前端已启动（见 README），Chrome/Edge 打开 http://localhost:5173，允许麦克风，点"🎤 开始聆听"。

## 一、基础能力（第 1 档，走规则快路径，<50ms）

| 说 | 预期 | 看点 |
| --- | --- | --- |
| 画一个红色的圆 | 中心出现红圆 | 规则快路径，指令日志标"规则"、延迟极低 |
| 画一个蓝色的大方块 | 蓝色大矩形 | 尺寸词"大"生效 |
| 在右上角画半径50的圆 | 右上角中等红圆 | **位置+数值尺寸也走规则快路径**（<50ms，非 LLM） |
| 撤销 | 方块消失 | 撤销 |
| 重做 | 方块回来 | 重做 |
| 清空 | 画布清空 | 清空 |

> 创作完成后说 **"导出"/"保存"** → 画布存为 PNG（"创作—修改—产出"闭环的产出环节）。

## 二、复杂指令拆解（第 2 档，走 LLM，约 1s）

| 说 | 预期 | 看点 |
| --- | --- | --- |
| 画三个绿色的圆排成一行 | 三个圆水平排列 | **一句话拆成 3 条指令**，日志标"LLM" |
| 在右上角画一个黄色三角形 | 三角形落在右上 | 九宫格定位 |
| 把刚才那个三角形放大一点 | 三角形变大 | **相对引用 + 修改** |

## 三、容错（准确性 + 容错性）

| 说（或模拟 ASR 错字） | 预期 | 看点 |
| --- | --- | --- |
| 画个圆圈 / 来个圆 / 整一个球 | 都画出圆 | L2 同义表达容错 |
| （识别成）画个原 | 触发澄清："我理解为画圆，对吗？" | **L3 澄清回问**，低置信度不硬画 |
| 是 | 执行画圆 | 语音确认 |

> 也可在"⚙ 同音词典"配置页现场加一条 `元→圆`，演示 L1 热更新纠错。

## 四、语义场景生成（第 2 档亮点，场景图分解 + 生图）

| 说 | 预期 | 看点 |
| --- | --- | --- |
| 画一只躺着的加菲猫，靠在黑色桌子边，桌上有一盆花 | 桌/猫/盆/花各就各位，先占位骨架→真图回填 | **场景图分解 + rel 空间落位 + 乐观渲染** |
| 把那盆花换成盛开的牡丹 | 只有花重渲染 | **多轮上下文 + 局部增量执行** |

## 五、QoS 面板

演示过程中右侧"QoS 指标"实时更新：规则快路径占比、平均/峰值延迟、澄清触发率——量化"响应延迟"与"容错"。
