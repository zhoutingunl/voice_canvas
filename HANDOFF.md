# VoiceCanvas 交付说明（提交/评审视角）

> 校园招聘项目试题：**纯语音控制的绘图工具**——用户不使用鼠标或键盘，仅通过语音指令完成绘图创作。
>
> 本文件面向**评审方**，直接回答试题要求的三件事：**计划支持哪些指令能力、最终实现了哪些、未完成部分的原因**。完整设计与细节见 [`design.md`](./design.md)，运行与演示见 [`README.md`](./README.md) 与 [`DEMO.md`](./DEMO.md)，过程见 GitHub PR #1–#11。

---

## 0. 30 秒了解

- **是什么**：语音说"画一个红色的圆""画三个圆排成一行""画一只加菲猫靠在黑桌边"，工具实时绘制到画布。
- **怎么实现的**：`语音 → ASR → L1 归一化 → 规则快路径 或 LLM(MiniMax-M3) 解析成指令 DSL → 画布渲染`；语义实体（猫/桌/花）经场景图分解 + 文生图（image-01-live）渲染。
- **怎么跑**：见 [`README.md` 快速开始](./README.md)（后端 Flask + 前端 Vite，三步起）；演示话术见 [`DEMO.md`](./DEMO.md)。
- **代码质量**：后端 36 + 前端 97 = **133 个单元测试全通过**；外部 API 全 mock、离线可跑。密钥仅存 `.env`、**绝不入库**。
- **协作方式**：Claude Code 辅助，人定方向、AI 落地、人工验收；全程 PR 提交（见 README「AI 协作说明」）。

## 1. 计划支持哪些指令能力（设计目标）

能力按三档规划（详见 `design.md §3`）：

**第 1 档 · 核心**：创建圆/矩形/椭圆/三角/直线/文字；颜色、尺寸（大/小/数值）、位置（九宫格+相对）；删除、清空、撤销、重做。

**第 2 档 · 进阶（亮点）**：批量+布局（一句拆多步，如"三个圆排成一行"）；相对引用/上下文指代（"把刚才那个往右移"）；修改已有图形（移动/缩放/改色/旋转）；**语义场景生成**（场景图分解 + 文生图，如加菲猫场景，并支持多轮增量修改）。

**贯穿能力**：指令理解 Hybrid（规则快路径+LLM）；三层容错（L1 归一化 / L2 LLM 容错 / L3 低置信度澄清）；低延迟（乐观渲染+占位+异步回填）。

**第 3 档 · 明确不做**：自由手绘曲线、纯文生图（不拆解直接出图）、图层/分组/批量导出、多人协同——原因见第 3 节。

## 2. 最终实现了哪些（对照计划）

> 详表与代码落点见 `design.md §17`。下面是面向评审的结论 + 证据。

| 计划能力 | 状态 | 证据 |
| --- | --- | --- |
| 第 1 档全部（创建/颜色/尺寸/位置/删除/清空/撤销/重做） | ✅ 已实现 | `canvasReducer` 快照式撤销重做；规则快路径实测<50ms |
| 第 2 档·批量+布局拆解 | ✅ 已实现 | 实测"画三个圆排成一行"→3 条 create 指令 |
| 第 2 档·相对引用/上下文 | ✅ 已实现 | parse 携带画布对象与 last_id |
| 第 2 档·修改图形（移动/缩放/改色/旋转） | ✅ 已实现 | update/move/transform |
| 第 2 档·语义场景生成（场景图分解+生图） | ✅ 已实现 | rel→坐标 resolver + image-01-live，实测加菲猫四实体落位 |
| 指令理解 Hybrid（规则+LLM 容错） | ✅ 已实现 | 实测"画个原"(同音错字)→圆，confidence 0.6 触发澄清 |
| 三层容错 L1/L2/L3 + 同音词典配置页 | ✅ 已实现 | normalize + 容错 prompt + 澄清横幅 + 可视化词典页 |
| 乐观渲染 + 占位骨架 + 异步回填 + 素材兜底 | ✅ 已实现 | entity 占位→真图；失败回退 emoji |
| ASR | ✅ 双引擎 | Web Speech（免费）+ **百炼 paraformer 云端**（稳定，PR #9） |
| 后端薄代理 + 密钥不入库 | ✅ 已实现 | Flask `/api/parse·imagine·asr`，key 仅在 `.env` |
| QoS 指标 | ✅ 面板 | 实时路径分布/平均·峰值延迟/规则占比/澄清率 |

**四个考点的兑现**：
- **指令理解准确性**：规则快路径 + MiniMax-M3 语义解析。
- **容错性**：三层防御（同音词典 + LLM 猜意图 + 低置信度澄清不硬画）。
- **响应延迟**：规则<50ms、乐观渲染秒级占位、真图异步回填；QoS 面板量化。
- **复杂指令拆解**：一句话→多原子指令；语义场景→场景图分解+关系坐标求解+增量执行。

## 3. 未完成部分及原因

> 详见 `design.md §18`。分两类：

### 受限未做（有后续路径）

| 项 | 原因 |
| --- | --- |
| Android 套壳编译为成品 APK | `android/` 已提供完整可构建的 WebView 套壳工程（麦克风授权 + 前台不锁屏，默认百炼 ASR）；编译为 APK 需 Android Studio/SDK，未在开发环境产出成品包 |
| 唤醒词 / 独立 VAD / TTS 回执 | 抗噪与语音回执为可选增强，非核心链路 |
| QoS 独立 Dashboard | 已有实时指标面板 + 指令日志；独立的历史聚合看板未做 |
| 文生图透明背景 | 图床返回普通图，未确认透明底；当前直接贴图、失败 emoji 兜底，未做前景抠图 |

### 主动不做（超范围 / 与考核目标不符）

| 项 | 原因 |
| --- | --- |
| 自由手绘曲线 | 纯语音难以精确描述坐标轨迹，投入产出比低 |
| 纯文生图（不拆解直接出图） | 会把"拆解/延迟/容错"外包给黑盒模型，**掩盖本题考核的工程能力**，是减分项 |
| 图层/分组/批量导出、多人协同 | 超出单机 demo 范畴 |

### 已知工程取舍（评审可关注）

- **ASR**：Web Speech 依赖 Google 服务，在国内网络时通时断（评审验收时实际遇到），故补充**百炼云端 ASR**作为稳定路径，界面可一键切换。
- **生图延迟**：文生图本身 2–5s，用"占位骨架→异步回填"消化，而非阻塞等待。
- **测试边界**：纯逻辑（解析/归一化/状态机/VAD/WAV/指标）单测覆盖；依赖真实麦克风音频与移动真机的环节由人工验收（开发者无麦克风环境）。

## 4. 给评审的验收指引

1. **跑起来**：按 [`README.md` 快速开始](./README.md)，桌面 Chrome/Edge 打开 `http://localhost:5173`，允许麦克风。
2. **按脚本走**：[`DEMO.md`](./DEMO.md) 按四个考点编排了话术与预期。
3. **网络注意**：若 Web Speech 不出字（Google 不可达），点界面"百炼（云端·稳定）"切换引擎。
4. **看证据**：右侧 QoS 面板（延迟/路径占比）、指令日志（规则/LLM/澄清）、加菲猫场景的占位→真图。
5. **查测试**：`cd backend && python3.11 -m pytest`（36）、`cd frontend && npm test`（91）。
